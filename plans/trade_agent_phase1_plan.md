# OmniEdge (全域工联) 基础设施规划书
## Phase 1：SOP 验证期 — 首个垂直品类切入

---

## 一、项目概述

OmniEdge (全域工联)是一个 AI 原生的跨境工业品自动化撮合平台。核心引擎是一个 15 节点的智能决策状态机，能够自主完成从买家询盘到订单履约的全自动化闭环。系统内置出口合规审查（RegGuard）、多轮 AI 谈判、背靠背套利对冲（Two-Phase Commit）、防篡改文档生成（DocuForge SHA-256）等金融级能力。

Phase 1 战略定位：以首个垂直品类作为切入点，通过真实跨境订单跑通标准化 SOP 闭环。首期场景特征为"低并发、高客单价、重合规"，天然适合验证 OmniEdge  的核心能力链——尤其是出口管制审查和防篡改合同生成。

验证成功标志：完成由 AI 全程主导的、从寻源到锁单到合规文档生成的完整跨境订单闭环。

---

## 二、系统架构

Phase 1 采用 **单节点本地推理 + 香港云端轻量部署 + 云端 API 借力** 的混合架构：

- **香港云端（单台）**：面向东南亚买家的业务入口，承载 Web 应用与数据库
- **本地 POC 验证节点（1 台工作站）**：运行核心 AI 模型，处理商业敏感推理任务
- **云端 API 辅助**：DeepSeek API 承担非敏感的寻源数据清洗，替代多机并行算力
- **加密隧道**：本地节点与香港云端通过 WireGuard VPN 互通

轻量化设计的核心逻辑：
1. 首期订单量有限，单线程串行高精度推理即可
2. 商业敏感数据（报价策略、客户画像、合规判断）保留在本地，不经过任何第三方
3. 非敏感的供应商寻源数据清洗交给 DeepSeek API（每百万 Token 约 ¥2），用极低 API 成本替代专用寻源机
4. 一次性硬件投入控制在 ¥28,000 以内，验证成功后再扩展

## 六、网络架构

### 6.1 网络拓扑

```
东南亚买家 (手机/PC)
        |
        v
Cloudflare CDN (全球加速 + WAF)
        | HTTPS
        v
腾讯云应用服务器 (香港)
        |
   -----+-----
   |         |
   v         v
PostgreSQL  Redis          <-- 香港云端
        |
   WireGuard VPN (加密隧道，千兆宽带)
        |
        v
   POC 验证节点             <-- 本地
   (RTX 4090 / qwen3:14b)
```

### 6.2 网络安全

| 措施 | 说明 |
|------|------|
| VPC 隔离 | 数据库和缓存仅允许应用节点内网访问，不暴露公网 |
| WireGuard VPN | 本地节点与香港云端之间的加密隧道 |
| Cloudflare WAF | OWASP 规则集 + Bot 防护 + IP 信誉过滤 |
| API 限流 | 全局 1,000 次/分钟 |
| 数据加密 | 全链路 AES-256-GCM |
| 硬件绑定 | 服务器硬件指纹绑定许可证 |

### 6.3 延迟预期

| 路径 | 延迟 |
|------|------|
| 东南亚买家 → 香港应用节点 | 20-60ms |
| 香港应用节点 → 本地 POC 节点 | 50-100ms (VPN) |
| 本地 AI 推理 (qwen3:14b) | 200-500ms |
| 端到端（买家询盘 → AI 响应） | <1.5s |
