# TradeForge 基础设施规划书
## Phase 3：全面扩张期 — 10,000+ 商户 / 全品类覆盖

---

## 一、阶段目标

在 Phase 2 规模化运营成功的基础上，Phase 3 进入全面扩张：

- 商户规模：从 3,000 家扩展到 **10,000-20,000 家活跃商户**
- 品类覆盖：**全品类工业品**（电子元件、AI 硬件、手机配件、工业传感器、通信设备等）
- 覆盖市场：东南亚四国 + **印度、中东、拉美** 等新兴市场
- 日均撮合量：**50,000-100,000 笔**
- 系统要求：多区域高可用 + GPU 推理集群 + 99.9% SLA

---

## 二、系统架构

Phase 3 升级为 **多区域云端集群 + 3 节点本地 AI 推理集群 + 弹性 API 层** 的全面扩展架构：

- **香港云端（4 台应用服务器 + 读写分离数据库）**：多节点负载均衡，数据库主从分离
- **本地 AI 推理集群（3 台高配工作站）**：全模型覆盖，分工协作，三重冗余
- **新增新加坡边缘节点**：降低印度/中东市场延迟
- **弹性 API 层**：DeepSeek + OpenAI + Claude 多供应商冗余

从 Phase 2 到 Phase 3 的关键升级：
1. 应用服务器从 2 台升级为 **4 台**，支撑 10,000+ 商户并发
2. 数据库升级为 **主从读写分离**，写入与查询互不干扰
3. 本地从 2 台升级为 **3 台 GPU 工作站**（含 2 张 RTX 5090），支撑全品类高并发 AI 推理
4. 新增 **新加坡边缘节点**，覆盖更多市场
5. Redis 升级为 **集群模式**，支撑万级 SSE 长连接

```

        v

   |         |
   v         v

   |         |
   +----+----+
        | VPC 内网 + 跨区域对等
   -----+-----------
   |    |           |
   v    v           v

        |

        |
   -----+------------------

        |

```

### 6.2 网络安全

| 措施 | 说明 |
|------|------|
| VPC 隔离 | 数据库/缓存/存储仅允许内网访问 |
| WireGuard VPN | 双专线冗余，主备自动切换 |
| Cloudflare WAF Business | 高级 OWASP 规则 + Bot 管理 + 自定义规则 |
| API 限流 | 每商户 60 次/分钟，全局 10,000 次/分钟 |
| 数据加密 | 全链路 AES-256-GCM + 传输层 mTLS |
| 硬件绑定 | 服务器硬件指纹绑定许可证 |
| 审计日志 | 加密审计日志 + 30 天保留 |

## 九、风险控制

| 风险 | 应对措施 |
|------|---------|
| 云服务器故障 | 4 节点负载均衡 + 新加坡边缘，多节点容灾 |
| 数据库故障 | 主从读写分离 + 每日备份 + 30 天回溯 + 跨区域备份 |
| 本地工作站故障 | 3 台三重冗余，任一台故障其余两台接管 |
| 本地断电 | 每台配备 UPS + 双专线（一主一备） |
| VPN 隧道中断 | 双专线自动切换 + 降级到多供应商 API |
| DDoS 攻击 | Cloudflare Business 高级防护 + 速率限制 |
| 数据泄露 | 全链路 AES-256 加密 + mTLS + VPC 隔离 + 硬件绑定 |
| 单供应商 API 故障 | 三供应商冗余（DeepSeek / OpenAI / Claude） |
| 区域性网络故障 | 香港 + 新加坡双区域部署 |
