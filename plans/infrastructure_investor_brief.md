## Phase 1: 市场验证期（0-6 个月）

### 商业目标
- 3,000-6,000 家东南亚商户入驻
- 日均 30,000 笔询盘撮合
- 覆盖越南、泰国、新加坡、马来西亚四国
- 主营品类：华强北电子元件 + AI 硬件 + 手机配件

### 技术架构概述

Project Claw 采用 **AI 原生架构**，核心引擎是一个 15 节点的智能状态机，能够自主完成从买家询盘到订单履约的全流程。系统内置出口合规审查、多轮智能谈判、背靠背套利对冲、防篡改文档生成等金融级能力。

与传统外贸 SaaS 的本质区别在于：每一笔交易都由 AI 智能体实时决策，而非人工操作。

本地部署大模型的战略意义：
1. **数据主权**：商业敏感数据（报价、客户画像、谈判策略）不经过第三方 API
2. **成本控制**：本地推理边际成本趋近于零，外部 API 按量付费仅作降级备用
3. **延迟优势**：本地推理 200-500ms vs 外部 API 1-3s

### 网络架构

采用混合云架构：香港云端面向全球用户，本地 AI 中心通过加密隧道提供推理算力。

- 买家访问路径：手机浏览器 → Cloudflare CDN → 香港应用节点 → AI 撮合引擎
- 商户实时推送：SSE 长连接，订单达成后毫秒级通知
- AI 推理路径：香港应用节点 → VPN 隧道 → 本地 GPU 推理 → 结果回传

### 合规与安全

- 全链路 AES-256-GCM 加密（客户信息、交易数据）
- 出口管制四层检查（制裁国、制裁港口、双用途物资、受限商品）
- SHA-256 防篡改哈希覆盖所有采购订单和发票
- UN 认证 + RCEP 零关税标签贯穿全流程

## Phase 2: 规模扩张期（6-18 个月）

### 商业目标
- 20,000-50,000 家商户
- 品类扩展至工业传感器、新能源组件、消费电子
- 覆盖东南亚全域 + 中东 + 拉美
- 日均 200,000+ 笔撮合

### 架构升级方向

从"双机热备"升级为 **Kubernetes 容器编排集群**，实现弹性伸缩。

| 层级 | Phase 1 | Phase 2 |
|------|---------|---------|
| 计算 | 2 台固定服务器 | K8s 集群 10+ 节点，自动扩缩容 |
| 数据库 | 单实例 | 主从读写分离 + 连接池 |
| 缓存 | 单节点 Redis | Redis 集群（48GB，3 主 3 从） |
| 消息 | 进程内事件总线 | RabbitMQ/Kafka 分布式消息队列 |
| AI 推理 | 本地单 GPU | 云端 GPU 集群（2× A100/L40S）+ 本地降级 |
| 监控 | 日志文件 | Prometheus + Grafana 全链路监控 |
| 灾备 | 无 | 新加坡异地灾备（自动故障切换） |

### AI 推理架构升级

Phase 2 的核心变化：AI 推理从本地单机迁移到云端 GPU 集群，本地工作站转为开发中心 + 紧急降级节点。

| 推理场景 | Phase 1 | Phase 2 |
|---------|---------|---------|
| 询盘解析 | 本地 qwen3:14b | 云端 vLLM 集群 (qwen3:32b) |
| 供应商寻源 | 本地 + DeepSeek API | 云端 GPU + 实时爬虫 |
| 谈判策略 | 本地 qwen3:14b | 云端 qwen3:32b |
| 视觉自愈 (VLM) | 本地 qwen-vl:7b | 云端 GPU 专用节点 |

```
Phase 1 (0-6月)          Phase 2 (6-18月)         Phase 3 (18月+)
¥6,500/月                ¥40,700/月               按需弹性
3K-6K 商户               20K-50K 商户              100K+ 商户
```
